# DIKSON-Li Wiki Architecture

## Цель

Wiki является долговременным контекстом проекта. Она хранит не только факты, но и решения, причины решений и связи между объектами.

## Структура

```
wiki/
├── projects/
├── sources/
├── decisions/
├── agents/
└── tasks/
```

## Принципы

1. Markdown как основной формат хранения.
2. Человек может редактировать данные вручную.
3. Агент может предлагать изменения, но не менять подтвержденные знания без контроля.
4. Каждая запись имеет источник и дату.

## Типы записей

- fact — подтвержденный факт;
- decision — принятое решение;
- hypothesis — рабочая гипотеза;
- task — задача;
- source — внешний источник.
