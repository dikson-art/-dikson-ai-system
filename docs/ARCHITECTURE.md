# Архитектура Dikson-Li

## Первый рабочий слой

Dikson-Li строится как локальная модельно-независимая система, где данные проекта принадлежат пользователю и хранятся в читаемых форматах.

Текущий слой включает:

- пакет `dikson_li`;
- конфигурацию через переменные окружения;
- append-only память проектов в JSONL;
- CLI-команды `remember` и `recall`;
- тесты базового хранилища.

## Следующие слои

1. Markdown Wiki с индексом связей.
2. Реестр агентов и skills.
3. OpenAI Responses API как интеллектуальный исполнитель.
4. Очередь задач и журнал запусков.
5. Локальный web-интерфейс.

## Принципы

- локальные данные по умолчанию;
- прозрачные текстовые форматы;
- воспроизводимые действия;
- отделение памяти от конкретной языковой модели;
- обязательное логирование изменений агентами.

## Пакетирование

Python-дистрибутив явно включает только исполняемые пакеты `app` и `dikson_li`. Каталоги локальных данных `data` и `wiki` не участвуют в package discovery и не попадают в wheel.
