# DIKSON AI System

Локальная проектная память и исследовательский помощник на OpenAI.

## Что уже работает в MVP

- создание изолированных проектов;
- хранение подтверждённой памяти и решений;
- загрузка PDF, DOCX, TXT и Markdown;
- разбиение документов на фрагменты;
- локальный поиск по материалам проекта;
- исследовательский ответ через OpenAI только на основании найденных фрагментов;
- возврат доказательной базы вместе с ответом;
- работа поиска без API-ключа.

## Быстрый запуск

Требуется Python 3.11 или новее.

```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
cp .env.example .env
uvicorn app.main:app --reload
```

Откройте интерактивную страницу API:

```text
http://127.0.0.1:8000/docs
```

## Первая проверка

1. В `POST /projects` создайте проект.
2. В `POST /projects/{project_id}/memory` сохраните правило или решение.
3. В `POST /projects/{project_id}/sources` загрузите PDF, DOCX или TXT.
4. В `GET /projects/{project_id}/search?q=...` проверьте поиск.
5. Добавьте `OPENAI_API_KEY` в `.env`.
6. В `POST /projects/{project_id}/research` задайте вопрос по загруженным материалам.

Пример запроса:

```json
{
  "question": "Как этот источник можно использовать в статье о книжном развороте?"
}
```

## Ограничения первой версии

- поиск пока лексический, без векторных представлений;
- данные хранятся локально в JSON;
- интерфейс пока представлен страницей FastAPI `/docs`;
- автоматическое обновление памяти требует отдельного подтверждаемого сценария;
- сканированные PDF без текстового слоя пока не распознаются.
